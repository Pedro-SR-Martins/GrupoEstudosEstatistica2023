Continuidade das anlises fatoriais

  • Os modelos bifactor so extenses da aplicao de anlise fatorial

  • Modelo bifactor no um modelo de duas dimenses

  • um tipo de aplicao que est em alta nas pesquisas recentes, por isso, importante que saibamos interpretar seu uso

Histrico

  • As ideias iniciais de modelo bifactor retomam os trabalhos clssicos de Spearman (1904) em relao ao fator g da inteligncia

  • \[ Desempenho = FatorG + FatorEs \] No caso do Spearman, havia uma tentativa de remover o erro de medida

  • Outra tradio de uso da anlise fatorial tem origem com os trabalhos de Thurstone, que tinha como lgica modelos de fatores correlacionados (\(~\) Dcada de 30)

  • As solues posteriores (nesse recorte) estavam relacionadas a modelos de segunda ordem e ao uso de anlise fatorial exploratria

  • Outras informaes podem ser vistas no captulo 18 do livro Hoyle (2023)

Diferena entre modelos bifactor e de segunda ordem

  • Possuem semelhanas e diferenas
  • Avaliam um escore geral
  • Diferenas sobre o efeito direto nos itens
  • Diferenas na estimativa de invarincia e para estimativas em modelos de SEM completos
  • Modelo bifactor pode ser utilizado a partir de 2 fatores especficos
  • Modelos de segunda ordem podem ser utilizados a partir de 2 fatores especficos aplicando restries extras (como fixando cargas fatoriais), MAS para que voc possa de fato avaliar os ndices de ajuste do modelo, necessrio ter pelo menos 4 fatores especficos

Para que serve um modelo bifactor?

  • Anlises bi-fatoriais so representaes de modelos de anlise fatorial em que um fator geral compete com os fatores de domnio especfico pela explicao da varincia dos itens

  • O fator geral, dessa forma, reflete uma dimenso que comum a todos os itens. Os fatores (domnios) especficos explicam a varincia que restante, ou seja, aquilo que no explicado pelo fator geral.

  • Como pode ser visto na imagem, em modelos bi-fatoriais as variveis latentes no so correlacionadas. Assume-se que toda a covariao existente entre os fatores especficos explicada pelo fator geral

Para que serve um modelo bifactor?

  • Modelos bi-fatoriais sugerem que a estrutura de um teste principalmente unidimensional, mas tambm explicada por fatores especficos. Dessa forma, no se encontra uma soluo fatorial para a escala nem utilizando um modelo com uma dimenso nem com as dimenses especficas. As duas devem coexistir.
  • Modelos bi-fatoriais podem revelar fraquezas dos domnios especficos de uma escala. Por exemplo, uma dimenso que no possui cargas fatoriais relevantes aps a incluso do fator geral pode ser um indicativo de que o fator especfico no to forte. Ou que as cargas fatoriais atribudas ao fator no so to relevantes ao construto.

Para que serve um modelo bifactor?

  • Modelos bi-fatoriais podem ser utilizados para:
    1. repartir a varincia entre o que do fator geral e dos fatores especficos;
    2. controlar para multidimensionalidade de medidas que so unidimensionais com nuances de varincia especfica;
    3. julgar se os itens possuem um fator geral forte o suficiente.
    4. determinar a adequao de um escore geral para uma escala e se vale a pena utilizar escores especficos das subescalas.

Fundamentos da anlise bifactor

  • A repartio da varincia possvel a partir de um modelo bifactor esta:
Fator geral || Fator especfico || Varincia especfica || Erro de medida |
  • No geral, no conseguimos separar a varincia especfica do item, ento ela tratada como erro (conceito de nuance de McCrae 2015 talvez ajude a entender isso)

Fundamentos da anlise bifactor

  • Dessa forma, a equao genrica da TCT para o desempenho \(X = T + E\) precisa ser expandida (temos duas fontes de escore verdadeiro)

  • Podemos estender para algo como:

  • \(X = C + U\)

  • Onde \(C\) contempla o fator geral + fator especfico e \(U\) o erro de medida e a varincia do nica do item

  • Isso implica que o item possui duas fontes de varincia sistemtica

  • \(X = T_{geral} + T_{especfico} + E\)

  • Isso justifica a necessidade de ndices especficos, portanto, para realizar uma avaliao precisa dos modelos

Problemas com a anlise

  • Com a popularidade do uso, aumenta o uso selvagem
  • Uma forma de refletir sobre o quo apropriado o uso das anlises bifactor est descrito no artigo

Problemas com a anlise

  • No basta usar os ndices de ajuste para justificar aceitar um modelo bifactor como o modelo final

    • Uma primeira inspeo deve estar relacionada ao padro de cargas fatoriais

Problemas com a anlise

  • importante ter uma teoria que embase o uso do fator geral. Ele, alm de forte, passvel de interpretao embasada?

  • Revisando:

    • O fator geral forte?

    • Quais fatores so confiveis?

    • Qual o padro de associao dos fatores com outras variveis?

ndices especficos para bifactor

mega hierrquico

  • Considerando a desvantagem da no separao da varincia do mega tradicional, a frmula do mega foi expandida para criar um coeficiente de mega hierrquico \(\omega_{H}\), especialmente vantajoso para contexto de modelos bi-fatoriais

  • O mega hierrquico contabiliza a varincia do fator geral tratando a varincia dos fatores especficos como erro

  • O coeficiente pode ser calculado facilmente utilizando a seguinte frmula (Rodriguez et al., 2016 equao 6):

  • \[\omega_{H} = {\frac{(\sum \lambda_{fator geral})} {(\sum \lambda_{fator geral}) + (\sum \lambda_{fator especfico1}) + (\sum \lambda_{fator especfico2}) + \sum (1 - h)}}\]

ndices especficos para bifactor

mega hierrquico

  • A variao do mega hierrquico o mega hierrquico subescala. Serve para calcular a confiabilidade dos fatores especficos

  • Neste caso, a varincia do fator geral tratada como erro (Rodriguez et al., 2016 equao 7)

  • \[\omega_{HS} = {\frac{(\sum \lambda_{fator especfico1})} {(\sum \lambda_{fator geral}) + (\sum \lambda_{fator especfico1}) + \sum (1 - h)}}\]

ndices especficos para bifactor

Construct reliability (replicability)

  • A confiabilidade do construto ou replicabilidade do construto pode ser confundida com a consistncia interna (porque recebe o mesmo nome de confiabilidade
  • Uma forma de evitar essa confuso pensar na replicabilidade do construto. O ndice \(H\) uma forma de se avaliar se a varivel latente bem representada pelo conjunto de itens utilizado (ou seja, a qualidade dos indicadores. Desta forma, uma varivel bem representada tende a ser mais replicvel entre os estudos. Valores de pelo menos .70 devem ser alcanados
  • O ndice \(H\) pode ser calculado usando a seguinte frmula (Rodriguez et al., 2016 equao 9):
  • \[H = 1/[1 + {\frac{1} {\sum_{i=1}^{k} {\frac{\lambda_{i}} {1 - \lambda_{i}}} }}]\]

Onde k o nmero de itens

ndices especficos para bifactor

Explained common variance (ECV)

  • \(ECV\) serve para julgar se os itens (ou conjunto de itens) so essencialmente unidimensionais
  • O \(ECV\) pode ser calculado com a seguinte frmula (Rodriguez et al., 2016 equao 10):

\[ECV = {\frac{(\sum \lambda_{fator geral}^2)} {(\sum \lambda_{fator geral}^2) + (\sum \lambda_{fator especfico1}^2)+ (\sum \lambda_{fator especfico2}^2)}}\]

  • Um \(ECV\) de 0.80 indica que o fator geral explica 80% da varincia comum aos itens, enquanto os fatores especficos explicam apenas 20% (dividido entre os fatores)
  • O \(ECV\) tambm pode ser calculado para os itens individuais, tentando identificar a porcentagem de varincia explicada pelo fator geral ao nvel dos itens. O \(ECV\) pode ser utilizado para criar uma escala mais unidimensional.
  • usando \(ECV-I\) para selecionar itens mais unidimensionais pode ser um boa ideia. para isso, sugere-se utilizar o ponto de corte de .80 ou .85

ndices especficos para bifactor

Percent uncontaminated correlations (ECV)

  • Basicamente, um indicador do quanto as correlaes entre as variveis observveis so influenciadas por um nico fator dividido pela quantidade de correlaes possveis

  • Indica, entre outras coisas, o quanto de informao voc perde ao forar uma soluo unidimensional a um conjunto de dados multidimensional

  • Quanto maior o valor de PUC, mais indicativo se tem de que a maioria das correlaes so informativas do fator geral
